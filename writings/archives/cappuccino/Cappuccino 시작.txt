Cappuccino 개요

### 개요

Cappuccino 는 브라우저 어플리케이션을 위한 웹 UI 프레임웍입니다.

웹 UI 프레임웍은 보통 JavaScript 로 만들어져 있지만 Cappuccino 는 특이하게도 Objective-J 로 만들어져 있습니다. API 는 Mac OS X Cocoa 와 유사합니다. 즉, Cappuccino 는 Mac App 을 만들던 개발자들이 Web App 개발을 용이하게 할 수 있도록 디자인된 프레임웍입니다. 반대로 말하면 JavaScript 를 사용하던 일반 웹 개발자들에게는 모든 것이 생소할 것입니다. 개발에 Mac 을 쓰지 않아도 여러가지 고통이 따르게 됩니다.

브라우저에서는 Objective-J 코드를 바로 실행할 수 없습니다. 그래서 Cappuccino 패키지에는 Objective-J to JavaScript 컴파일러가 따라옵니다. 크기는 현재 92KB 정도로 Cappuccino 패키지에서 유일하게 JavaScript 로 되어 있습니다. 프래임웍 코드들은 브라우저에 로딩되어 JavaScript 로 컴파일된 후 실행됩니다. 

Cappuccino 는 서버 사이드 JavaScript 실행환경 Narwhal 을 포함하고 있습니다. 웹 개발을 위해 Cappuccino 를 사용하는 사람들은 사용할 일이 없겠으나 알아두는 것이 패키지 구조를 이해하는데 도움이 됩니다. Narwhal 은 느리고 기능이 부족해서 범용으로 쓸만큼 좋지는 않습니다. 이 부분의 대세는 Node 겠지요. Narwhal 에는 패키지 매니저 Tusk, Objective-J 빌드툴 Jake 도 따라옵니다. 이것도 크게 신경쓸 필요는 없습니다.


### 설치

Github 첫 페이지에 설치법이 나와있습니다.

[https://github.com/cappuccino/cappuccino/](https://github.com/cappuccino/cappuccino/)

버전마다 많이 달라지는데 0.9.5 경우는 아래와 같았습니다.

	$ curl https://raw.github.com/cappuccino/cappuccino/v0.9.5/bootstrap.sh >/tmp/cappuccino_bootstrap.sh && sh /tmp/cappuccino_bootstrap.sh

설치 스크립트는 /usr/local/narwhal 아래에 관련 파일들을 설치합니다. 이미 디렉토리가 존재하면 삭제할 것인지 선택할 수 있습니다. Mac 의 경우 Java 로 구현된 JavaScript 엔진 대신 Native JavaScript Core 를 사용할 선택권이 주어집니다. 무조건 yes 합니다. 설치 말미에 `/usr/local/narwhal/bin` 을 `PATH` 에 추가하도록 쉘 초기화 스크립트를 수정할 것인지 뭍는데 yes 합니다. Cappuccino 프록젝트를 생성하려면 코맨드라인에서 `capp` 명령을 써야하기 때문입니다. CAPP_BUILD 에 대한 안내도 나오는데 일반 사용자는 무시합니다. 설치가 끝났으면 쉘을 재기동 하기위해 터미널 창을 새로 만듭니다.


### Hello World

터미널에서 원하시는 디렉토리로 가서 아래 명령을 실행합니다.

	capp gen --symlink hello

`hello` 라는 디렉토리가 생성되고 기본 템플릿들이 복사됩니다. `--symlink` 옵션은 Cappuccino 프레임웍 파일들을 복사하지 않고 심볼릭 링크를 사용한다는 의미입니다. 프로젝트마다 같은 파일들이 복제되면 SSD 가 너무 아깝습니다.

사파리를 띄워서 `index.html` 파일을 열어봅니다. 퍼미션 문제로 크롬에서는 로컬 테스트할 수 없습니다.
